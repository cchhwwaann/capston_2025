🤖 굴삭기 버킷 효율 분석 캡스톤 프로젝트 요약 (v2)
1. 🎯 프로젝트 목표
*AI 비전 카메라(OAK-D Lite)**와 로드셀 센서를 활용하여 굴삭기 모형의 작업을 실시간으로 분석.
작업 단계를 [굴착, 이송, 투하, 대기] 4가지 상태로 자동 분류.
'사이클 타임'과 '유효 작업량(Payload)'을 측정하여 버킷의 최종 효율을 계산.
2. 🔩 하드웨어(H/W) 구성 계획 (수정됨)
기구 제작 (Mechanical)
스케일: 1/14 스케일의 대형 모델 (붐 길이 약 40cm).
액추에이터: 실제 유압 실린더 역할을 하도록 12V DC 리니어 액추에이터 3개 (붐, 암, 버킷용)를 사용.
관절 (힌지 핀): 부드러운 회전을 위해 규격화된 힌지 핀을 사용.붐/암/버킷 연결부: 8mm 힌지 핀
액추에이터 연결부: 4mm 힌지 핀
센서 및 비전 (Sensing)
3D 비전: OAK-D Lite 카메라RGB 영상과 깊이(Depth) 맵을 동시에 사용.
버킷의 2D 위치(YOLO)와 깊이 정보를 조합하여 **3D 공간 좌표(X,Y,Z)**를 실시간으로 획득.
하중 측정: 로드셀 (또는 스트레인 게이지) + HX711 앰프암(Arm)에 부착하여 버킷에 담긴 흙의 **실제 무게(Payload)**를 측정.
이 데이터는 '유효 작업'을 판단하는 핵심 기준이 됨.
전원 및 제어 (Power & Control)
제어기: **아두이노(Arduino)**를 메인 제어기로 사용.3단 스위치 3개를 아두이노의 **디지털 입력(Input)**으로 연결.
아두이노가 스위치 신호(전진/정지/후진)를 읽어, **모터 드라이버(Motor Driver)**를 통해 3개의 액추에이터를 개별적으로 제어.
전원: 12V 20A (240W) SMPS3개의 액추에이터가 동시에 동작할 때 필요한 높은 전류를 안정적으로 공급하기 위해 고용량 산업용 파워서플라이를 구매하여 사용.
3. 🖥️ 소프트웨어(S/W) 및 AI 계획
개발 환경 (Environment)
메인 플랫폼: Python 3.x
코드 편집기: VS Code (Visual Studio Code)
환경 관리: Anaconda (Conda)OAK-D, 텐서플로우, OpenCV 등 수많은 라이브러리 간의 버전 충돌을 막기 위해 '가상환경'을 생성하여 프로젝트를 관리.
4단계 개발 로드맵
데이터 수집 (Data Acquisition)[PC/Python]: DepthAI 라이브러리로 OAK-D 카메라 제어. PySerial 라이브러리로 아두이노와 시리얼 통신.
[MCU/Arduino]: HX711 라이브러리로 로드셀 무게를 측정. 이 무게 값을 PC로 Serial.println()을 통해 실시간 전송. (액추에이터 제어 코드와 통합)
특징 추출 (Feature Extraction)[YOLO 비전]: Roboflow (웹 플랫폼)를 사용해 버킷/암 이미지를 학습, 커스텀 YOLO 모델 생성.
[3D 좌표]: OAK-D에서 YOLO를 실행, 탐지된 버킷의 2D 위치 + 깊이 맵을 조합하여 버킷 3D 좌표 (X,Y,Z) 획득.
AI 동작 분류 (AI Classifier)[입력 데이터]: 시간 순서에 따른 (1) 버킷 3D 좌표, (2) 역기구학으로 계산된 붐/암 각도, (3) 필터링된 로드셀 하중 값.
[AI 모델]: 시계열 데이터 처리에 강한 RNN (LSTM/GRU) 모델을 TensorFlow/Keras로 학습.
[출력]: 현재 상태가 [굴착, 이송, 투하, 대기] 중 무엇인지 실시간으로 분류.
효율 분석 (Analysis & App)AI가 분류한 상태를 기반으로, 사이클 타임 (굴착 시작 ~ 투하 완료)과 유효 작업량 (이송 시 하중)을 계산.
최종 **버킷 효율 (g/sec)**을 도출하여 화면에 표시.
